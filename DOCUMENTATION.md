# QuestBook — Документация

## Что это

Приложение для геймификации личных целей с системой бадди-accountability.
Пользователь создаёт квесты (задачи) и награды, зарабатывает коины за выполнение квестов и тратит их на награды. Бадди видит твой прогресс — это создаёт внешний контроль и мотивацию.

---

## Стек

| Слой | Технология | Назначение |
|------|-----------|------------|
| Фронтенд | React + Vite | UI приложения |
| Стили | Чистый CSS | RPG тёмная тема |
| База данных | Supabase (PostgreSQL) | Хранение данных |
| Авторизация | Supabase Auth | Email/password |
| Хостинг | Vercel | Деплой фронтенда |
| Уведомления | Telegram Bot API (планируется) | Push через Telegram |

---

## Архитектура

```
Пользователь
     ↓
Vercel (React SPA)
     ↓
Supabase (Auth + PostgreSQL + Edge Functions)
     ↓
Telegram Bot API (планируется)
```

Фронтенд общается с Supabase напрямую через JS клиент. Бэкенд-сервера нет — вся логика либо на клиенте, либо в Supabase (RLS политики, триггеры, функции).

| created_at | timestamp | |

**rewards**
| Колонка | Тип | Описание |
|---------|-----|----------|
| id | uuid | первичный ключ |
| user_id | uuid | → profiles.id |
| title | text | название награды |
| coins | integer | стоимость в коинах |

| Колонка | Тип | Описание |

## Безопасность (RLS политики)

Каждая таблица защищена Row Level Security. Пользователь видит только своё, бадди видит профиль, квесты, награды и лог друг друга.

| Таблица | Политика |
|---------|----------|
| profiles | своё + профили бадди + поиск по invite_code (all: true) |
| quests | своё + квесты бадди (select) |
| rewards | своё + награды бадди (select) |
| logs | своё + лог бадди (select) |
| buddies | связи где участник = текущий пользователь |

---

## Серверная логика

**Триггер `on_auth_user_created`** — при регистрации нового пользователя автоматически создаёт запись в `profiles` с username из email.

**Функция `add_coins(p_user_id, p_amount)`** — атомарно обновляет баланс коинов в profiles. Вызывается через `supabase.rpc()` при выполнении квеста (+coins) и взятии награды (-coins).

---

## Флоу пользователя

```
Регистрация → автосоздание профиля (триггер)
     ↓
Dashboard: квесты / награды / лог / бадди
     ↓
Выполнил квест → запись в logs + add_coins(+) → обновление баланса
Взял награду  → проверка баланса → запись в logs + add_coins(-) → обновление баланса
     ↓
Бадди: поделился invite_code → друг добавил по коду → видят профили друг друга
```

## Планируется

- **Telegram уведомления** — при выполнении квеста бадди получает сообщение в Telegram. Реализация через Supabase Edge Functions + Telegram Bot API. В таблице buddies колонка `notifications_enabled` для управления уведомлениями от конкретного бадди.
- **Модальные окна** — замена `prompt()` на нормальные формы для добавления квестов и наград.
- **Google авторизация** — дополнительный провайдер через Supabase Auth.
- **AI генератор квестов и наград** — персонализированные шаблоны через Claude API.
